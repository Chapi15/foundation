ERPNext User Activities

  • Issue Cleanup Sprint: Review and respond to un-replied issues. Review all old issues and either tag them or close them if they are not relevant.
  • Documentation Sprint: Writing missing documentation for ERPNext modules or features.
  • Article Writing Sprint: Write general purpose articles on ERP systems, benefits, challenges, implementation strategies that can be published on ERPNext.org

Developer Activities

  • Bug quashing sprint: Try and fix all issues tagged as “bug” on GitHub Issues.
  • Build a feature: Scratch your own itch by contributing a feature you really want in the product.
  • Test Coverage Sprint: Add test coverage, refactor tests where missing.
  • Documentation Sprint: Add missing developer documentation.
